# The Circle — Plovdiv (live + редактируемо)

Приложение за откриване на събития, заведения и барове в Пловдив.
Данните живеят в **Supabase** (безплатна база) и се управляват от вграден **админ панел**.
Хоства се безплатно на **Vercel**.

---

## Какво ще постигнеш
- Приложение на истински адрес (напр. `the-circle.vercel.app`, после и собствен домейн)
- Админ панел на `твоят-адрес/?admin` — добавяш/редактираш/триеш места и събития от форма, без код
- Промените се виждат от всички веднага (обща база)

---

## Стъпка 1 — Supabase (базата данни) ~5 мин
1. Влез в https://supabase.com → **Start your project** (безплатно, с Google/GitHub).
2. **New project** → дай име (`the-circle`), парола за базата, регион Frankfurt. Изчакай ~2 мин.
3. Отляво → **SQL Editor** → **New query**.
4. Отвори файла `supabase_schema.sql` от този проект, копирай **целия** и го постави в редактора → **Run**.
   Това създава таблиците `venues` и `events` и налива началните места.
5. Отляво → **Project Settings → API**. Запиши си две неща:
   - **Project URL**
   - **anon public** ключ

## Стъпка 2 — Качи кода в GitHub ~3 мин
1. Направи акаунт в https://github.com (ако нямаш).
2. Създай нов **repository** (напр. `the-circle`), private или public.
3. Качи всички файлове от тази папка (бутон **Add file → Upload files**, или с Git).
   `node_modules` и `dist` НЕ се качват (вече са в `.gitignore`).

## Стъпка 3 — Deploy на Vercel ~5 мин
1. Влез в https://vercel.com с GitHub акаунта.
2. **Add New → Project** → избери repository-то `the-circle` → **Import**.
3. Vercel сам разпознава Vite. Преди да натиснеш Deploy, отвори **Environment Variables** и добави трите:
   | Name | Value |
   |------|-------|
   | `VITE_SUPABASE_URL` | Project URL от Supabase |
   | `VITE_SUPABASE_ANON_KEY` | anon public ключ |
   | `VITE_ADMIN_PASSWORD` | твоя парола за админ (смени я!) |
4. **Deploy**. След ~1 мин получаваш линк. Готово — приложението е live.

## Стъпка 4 — Добавяй места
- Отвори `твоят-адрес/?admin`
- Въведи паролата (`VITE_ADMIN_PASSWORD`)
- Таб **Обекти** или **Събития** → **+ Нов запис** → попълни → **Запази**
- Промяната отива директно в базата и се вижда в приложението при опресняване

> При добавяне на събитие, полето **ID на обекта** трябва да съвпада с `id` на вече съществуващ обект (виж списъка в таб „Обекти").

## Локално стартиране (по желание, за тестване на компютъра)
```bash
npm install
cp .env.example .env      # попълни стойностите
npm run dev               # отваря на http://localhost:5173
```

## Бележки за сигурност
- SQL-ът засега позволява писане с анонимния ключ (за да работи админът лесно).
  Това е ОК за старт. За по-сериозна сигурност по-късно се включва Supabase Auth и
  политиките за писане се ограничават само до логнат потребител.
- Картата в детайлите е статична визуализация; за интерактивна карта по-късно се
  добавя Leaflet/Mapbox.

## Структура
```
src/
  App.jsx            UI на приложението (екрани, навигация)
  Admin.jsx          админ панел (CRUD)
  data.js            начални (seed) данни
  supabaseClient.js  връзка с базата
supabase_schema.sql  таблици + начални данни за Supabase
.env.example         примерни ключове
```
